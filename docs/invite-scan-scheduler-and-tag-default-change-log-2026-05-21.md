# WeFlow 活动标签与定时扫描调整变更记录（2026-05-21）

## 已修改内容

1. WeFlow-main 发售群列表活动标签展示
   - 原问题：发售群列表也显示“全部活动”，但这里的口径本质是多个标签的聚合，不应当被当成一个真实标签。
   - 修改方式：仅在发售群列表页隐藏“全部活动”选项，其他页面保持原样。
   - 说明：发售群列表仍保留聚合展示，不影响列表数据本身。

2. WeFlow-main 检查是否退出群定时频率
   - 原问题：首次执行时间过早。
   - 修改方式：改为启动后第 5 分钟触发一次，之后每 1 小时触发一次。
   - 手动按钮：保留不变。

3. WeFlow-main 增量扫描定时频率
   - 原问题：首次和后续间隔不符合新要求。
   - 修改方式：改为启动后第 3 分钟触发一次，之后每 30 秒一次。
   - 扫描范围：继续使用所有启用活动标签下已绑定的群，重复规则不变。

4. WeFlow-Web 数据大屏默认标签
   - 原问题：进入页面后默认停在“全部活动”。
   - 修改方式：优先默认选择名称为“拉新”的活动标签。
   - 回退：如果没有“拉新”，继续使用原有默认逻辑。

## 已验证

- `WeFlow-main` `npm run typecheck`
- `WeFlow-Web` `npm run typecheck`
- `WeFlow-main` `npx vite build`
- `WeFlow-Web` `npm run build`

## 剩余说明

- 发售群列表当前通过隐藏“全部活动”来避免把聚合口径包装成可选标签；聚合视图本身仍然存在。
- 自动扫描频率改为 30 秒后，后台任务会更频繁，现有并发保护逻辑保持不变。
