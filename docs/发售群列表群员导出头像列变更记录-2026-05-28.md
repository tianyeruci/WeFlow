# WeFlow-Web 发售群列表群员导出头像列变更记录

## 变更内容
- 将发售群列表中的行内 `导出群员` 从 CSV 改为 XLSX。
- 将 `批量导出所有群员` ZIP 内的单群文件从 CSV 改为 XLSX。
- 群员导出列顺序调整为：
  - `时间`
  - `邀请人`
  - `头像`
  - `被邀请人`
  - `状态`
- `头像` 列使用被邀请人的头像，不使用邀请人的头像。
- 头像来源复用现有成员事件字段：
  - `head_img`
  - `avatar_url`
  - `raw_json.head_img`
  - `raw_json.avatar_url`
  - `raw_json.avatarUrl`
- 服务端生成 XLSX 时下载头像并嵌入单元格；头像为空、下载失败、超时、过大或格式不支持时，该单元格留空，不阻断导出。
- 未新增数据库字段，未新增 npm 依赖。

## 主要修改文件
- `WeFlow-Web/lib/invite-data.ts`
  - 群员导出数据由二维 CSV 行改为结构化行。
  - 新增导出行字段 `avatarUrl`。
  - 批量导出文件名从 `.csv` 改为 `.xlsx`。
- `WeFlow-Web/lib/xlsx.ts`
  - 新增最小 XLSX 生成逻辑。
  - 生成工作簿、工作表、drawing、media、relationships 等 XLSX 必要文件。
  - 将头像图片锚定到 `头像` 单元格内。
- `WeFlow-Web/lib/zip.ts`
  - `buildZipArchive` 新增 `preservePaths` 选项，供 XLSX 内部目录结构使用。
  - 默认行为保持原有 ZIP 文件名清洗逻辑，不影响普通批量 ZIP。
- `WeFlow-Web/app/api/invite/export/groups/route.ts`
  - `mode=member` 响应改为 XLSX。
  - `mode=batch` 仍响应 ZIP，但 ZIP 内部文件改为 XLSX。
  - `summary` 和 `list` 继续保持 CSV。
- `WeFlow-Web/app/page.tsx`
  - 单群 `导出群员` 的浏览器兜底下载文件名从 `.csv` 改为 `.xlsx`。

## 接口变化
- `/api/invite/export/groups?mode=member`
  - 原响应：`text/csv; charset=utf-8`
  - 新响应：`application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
  - 文件扩展名：`.xlsx`
- `/api/invite/export/groups?mode=batch`
  - 外层仍为 `application/zip`
  - ZIP 内单群文件扩展名由 `.csv` 改为 `.xlsx`
- `/api/invite/export/groups?mode=summary` 和 `mode=list` 不变。

## 验证结果
- `npm run typecheck`：通过。
- 运行内存 XLSX 生成探针：通过。
  - 确认生成内容包含 `[Content_Types].xml`。
  - 确认生成内容包含 `xl/worksheets/sheet1.xml`。
  - 确认生成内容包含 `xl/drawings/drawing1.xml`。
  - 确认生成内容包含 `xl/media/image1.png`。

## 后续人工验收建议
- 在 WeFlow-Web 页面进入 `发售群列表`。
- 点击单群行内 `导出群员`，确认下载文件为 `.xlsx`，且 `邀请人` 和 `被邀请人` 中间存在 `头像` 列。
- 打开 XLSX，确认头像显示在对应成员行的 `头像` 单元格内。
- 点击 `批量导出所有群员`，确认 ZIP 内每个群员文件为 `.xlsx`，列结构与单群导出一致。
