# 发售群列表执行计划（WeFlow-Web）

## 目标
在 `WeFlow-Web` 现有单页壳中新增 `发售群列表` 视图，继续按当前活动标签过滤数据，只实现图六勾选部分的只读和导出能力。

## 范围
- 保留现有 `数据大屏` 和 `群成员溯源`
- 新增 `发售群列表`
- 只做以下能力：
  - `包含已退群的人` 开关
  - 顶部群数量 / 人数摘要
  - `合计导出群人数`
  - `批量导出所有群员`
  - `合计导出群列表`
  - 行内 `导出群员`
- 不增加搜索、排序、标签管理、扫描、编辑、删除等能力

## 实施步骤
1. [落文档] -> verify: 本文件已保存到 `docs`
2. [扩展页面视图] -> verify: 顶部可切换 `数据大屏 / 发售群列表 / 群成员溯源`
3. [补群列表数据展示] -> verify: 群列表按当前活动标签展示，能看到群名称、人数、操作按钮
4. [补导出接口] -> verify: 四个导出入口都返回正确文件类型和文件名
5. [补样式与说明] -> verify: 页面布局、下载态、README 描述同步更新
6. [校验构建] -> verify: `npm run typecheck` 与 `npm run build` 通过
7. [补变更记录] -> verify: 变更清单已保存到 `docs`

## 验收标准
- 切换活动标签后，群列表和顶部人数摘要同步刷新
- `包含已退群的人` 只影响顶部总人数口径，不改表格行内人数
- `合计导出群人数`、`合计导出群列表`、`导出群员` 都能下载 CSV
- `批量导出所有群员` 下载 ZIP，ZIP 内每个群一个 CSV
- 数据大屏、群成员溯源、已有导出行为不回归

## 风险与处理
- ZIP 打包需要保证兼容浏览器下载和服务器端响应
- 群列表排序口径以当前活动标签下的群绑定数据为准
- 若导出字段还需微调，以截图确认口径为准，优先保持只读和导出范围不扩张
