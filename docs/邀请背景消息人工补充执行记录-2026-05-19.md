# 邀请背景消息人工补充执行记录-2026-05-19

## 目标

- 识别 `A邀请你加入了群聊，群聊参与人还有：B、C、D` 这类背景系统消息时，只生成一条 `A -> 你` 的待确认邀请记录。
- 在待确认页支持人工补充有效信息和人工拆分新增邀请记录。
- 新增记录必须进入现有 `inviteEvents -> rawEvents -> recomputeFlags -> dashboard/ranking/trace` 链路，并参与统计。

## 实际改动

- 后端 `inviteStatsService`：
  - 新增背景邀请解析规则，保留 `source_rule` 和 `source_context_members`。
  - 待确认确认逻辑支持补 `groupId`、`wxId`、`inviterWxId`，并校验群是否属于当前活动标签。
  - 新增 `addManualInviteRecord`，可把待确认记录拆成一条真实的确认邀请记录。
  - 根据 wxid 解析群成员/联系人头像，回填 `head_img`。
  - 原始事件与物化事件同步时保留 `head_img`、`source_rule`、`source_context_members`。

- 前端待确认页：
  - “有效”改为先弹补充表单。
  - 新增“添加信息”入口，可从背景消息中的成员列表快速补一条真实邀请记录。
  - 保存后刷新待确认、成员溯源和大屏。

- 类型与 IPC：
  - 更新 `confirmPending` payload。
  - 新增 `addManualInviteRecord` IPC。

- Supabase 视图：
  - `final_stat_events` 补出 `raw_json` 和 `head_img`，继续兼容现有头像读取逻辑。

## 残余风险

- 背景消息的正则覆盖的是当前描述格式，若微信文案再变体，可能需要补解析分支。
- 远端头像依赖 `raw_json.head_img` / `avatar_url` 字段可见性，视图已补齐，但远端数据同步链路若未刷新旧数据，旧记录头像仍可能为空。
- “添加信息”会要求手工补齐 wxid，若群成员 wxid 不在本地群成员数据里，头像只能回落到联系人查询结果。
