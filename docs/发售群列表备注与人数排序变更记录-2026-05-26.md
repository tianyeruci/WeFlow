# WeFlow-Web 发售群列表备注与人数排序变更记录

## 变更内容
- 新增 `supabase/web_group_remarks_2026_05_26.sql`：
  - 创建 `web_group_remarks` 表。
  - 主键为 `account_scope, group_id`，确保运营备注绑定群 ID，不绑定群名称。
  - 新增 `group_id`、`updated_at` 查询辅助索引。
  - 授权 `service_role` 对表执行 `select, insert, update, delete`。
- 新增 `REMOTE_REMARK_TOKEN` 写入口令：
  - 仅 `PUT /api/invite/group-remarks` 校验。
  - 普通读取、页面查看和导出不受影响。
- 发售群列表新增“备注”列：
  - 行内输入，失焦或回车保存。
  - 首次保存弹窗输入写入口令并存入浏览器 `localStorage`。
  - 保存失败时保留当前输入并展示错误。
- 发售群列表人数列新增升序/倒序排序：
  - 默认保持原有顺序。
  - 点击人数表头后先按人数倒序，再次点击切换升序。
  - “合计导出群列表”同步当前人数排序。
- “合计导出群列表”新增 `群备注` 列：
  - CSV 表头调整为 `群名称, 群ID, 群备注, 人数`。
  - 单群成员导出、批量 ZIP、人数汇总不新增备注列。
- README 已补充 `REMOTE_REMARK_TOKEN` 和 `web_group_remarks` 数据要求说明。

- `npm run typecheck`：通过。
- `npm run build`：通过。
- 首次 `npm run build` 在普通沙箱中因 `spawn EPERM` 失败，已按权限规则在授权环境下重跑并通过。

## 部署注意
- 部署前需要先在 Supabase 执行 `supabase/web_group_remarks_2026_05_26.sql`。
- 生产环境需要配置 `REMOTE_REMARK_TOKEN`，否则备注保存接口会返回配置错误。
