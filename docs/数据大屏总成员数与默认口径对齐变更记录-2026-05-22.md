# 数据大屏总成员数与默认口径对齐变更记录

## 修改内容

- 保存执行计划：
  - 新增 `docs/数据大屏总成员数与默认口径对齐执行计划-2026-05-22.md`。

- Web 默认口径：
  - `WeFlow-Web/lib/invite-data.ts`
    - `totalMembers` 保持按当前活动标签下绑定群的 `member_count` 合计。
    - `totalMembersWithQuit` 改为 `totalMembers + 退出状态溯源行数 + 今日退出人数`。
    - 今日退出人数改为按成员 key 统计，减少同一成员同时存在退群事件和邀请退群标记时的重复。

- Main 默认口径：
  - `WeFlow-main/electron/services/inviteStatsService.ts`
    - `getDashboard` 中 `dedupeMembers` 改为仅显式传入 `true` 时启用去重，默认不去重。
    - 默认未去重时，总成员数使用活动标签下绑定群 `member_count` 合计。
    - 默认未去重且勾选“含退群”时，按 `总成员数 + 退出状态溯源行数 + 今日退出人数` 计算。
    - Main 去重开关开启后，继续沿用原去重分支。
    - 邀请排行榜构建默认改为不去重，和 Web 默认一致。
    - 邀请排行榜导出未传 `dedupeMembers` 时默认不去重。

## 验证结果

- `WeFlow-main`: `npm run typecheck` 通过。
- `WeFlow-Web`: `npm run typecheck` 通过。
- 根目录：`git diff --check` 通过。
- `git diff --check` 仅提示相关文件下次由 Git 转成 CRLF，没有空白错误。

## 手工核对建议

- Web 与 Main 默认未勾选去重时，总成员数一致。
- 总成员数 = 当前活动标签下所有绑定群 `member_count` 合计。
- 勾选“含退群”后 = 总成员数 + 溯源 quit 状态人数 + 今日退出人数。
- Main 勾选去重后仍按原去重逻辑表现。
- Main/Web 默认邀请排行榜人数一致，并与非去重邀请溯源口径一致。
