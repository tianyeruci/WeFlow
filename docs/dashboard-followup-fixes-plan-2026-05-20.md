# 数据大屏与邀请统计后续修复执行计划（2026-05-20）

## 目标

在不改表结构、不引入新依赖的前提下，修复 WeFlow-main 与 WeFlow-Web 数据大屏、远程同步设置、待确认补充弹窗、退出群定时器、发售群列表的展示与行为问题。

## 关键假设

- “恢复初始化”弹框标题不能继续依赖浏览器原生确认框，改为 WeFlow-main 页面内自定义确认弹窗。
- 群人数进度条的 100% 基准为 `max(500, 当前范围内最大群人数)`。
- “下载图片布局不美观”主要指排行榜筛选栏中的下载按钮挤压和溢出，优先通过收紧工具栏布局修复。
- WeFlow-main 发售群列表在“全部活动”下展示所有已绑定活动标签的群，不绑定到某个单一活动标签。
- 检查是否退出群的手动按钮逻辑保持不变，只调整自动定时频率。

## 执行步骤

1. 修复 WeFlow-main 远程同步设置恢复初始化确认弹窗
   - 用页面内自定义确认弹窗替换 `window.confirm`。
   - 弹窗标题显示“恢复初始化”。
   - verify: 搜索确认不再通过该恢复入口调用原生 `window.confirm`。

2. 修复群人数展示进度条基准
   - WeFlow-main 和 WeFlow-Web 均使用 `max(500, 最大群人数)` 作为进度条满格基准。
   - verify: 199 人且无群超过 500 时宽度约 40%，500 人为满格。

3. 优化邀请人数排行榜工具栏与下载图片按钮布局
   - WeFlow-main 和 WeFlow-Web 收紧排行榜工具栏布局，避免下载图片按钮挤压、错位或溢出。
   - verify: 宽屏和窄屏下筛选项与按钮排列稳定。

4. 修复 WeFlow-main 邀请人数排行榜柱形图颜色
   - WeFlow-main 柱形图使用与 Web 端一致的多色条形。
   - verify: 排行榜柱形图不再只有单一绿色。

5. 修改排行榜说明文案
   - WeFlow-main 和 WeFlow-Web 将“有效入群人数”改为“入群人数”。
   - verify: 搜索页面源码无该处旧文案。

6. 修复补充有效邀请复制按钮
   - WeFlow-main 补充有效邀请弹窗中的复制按钮只显示图标，保留 `title` / `aria-label`。
   - verify: 弹窗中复制按钮不再出现“复制”文字。

7. 调整退出群检查定时任务
   - WeFlow-main 启动 1 分钟后执行第一次检查，之后每 1 小时执行一次。
   - 手动触发按钮保持不变。
   - verify: 定时器间隔常量为 60 分钟。

8. 修复 WeFlow-main 发售群列表“全部活动”口径
   - 选择“全部活动”时展示所有活动标签下的群。
   - verify: 发售群列表不再因选中全部活动而显示 0 个微信群。

9. 验证与记录
   - 运行 WeFlow-main typecheck / build。
   - 运行 WeFlow-Web typecheck / build。
   - 将最终改动、原问题、处理方式和剩余隐患记录到本地变更日志。
