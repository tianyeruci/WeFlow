# 群成员溯源导出补充原始消息变更记录

日期：2026-05-29

## 背景

WeFlow-main 的“群成员溯源”页面表格已经展示“原始消息”，字段来源为 `InviteMemberTraceRow.raw_content`。但本地导出调用 `InviteStatsService.exportMemberTrace` 时，导出表头和行数据只到“类型”列，没有写出 `raw_content`，导致导出文件缺少原始消息。

## 本次修改

- 修改 `WeFlow-main/electron/services/inviteStatsService.ts`
  - 在 `exportMemberTrace` 的导出表头末尾新增“原始消息”列。
  - 在导出行数据末尾新增 `row.raw_content`，与页面展示使用同一字段。

## 未修改内容

- 未修改群成员溯源页面展示、筛选、分页、排序逻辑。
- 未修改扫描、归因、待确认、原始事件导出逻辑。
- 未修改 WeFlow-Web 远程导出链路；该链路已在 `getMemberTraceExportRows` 中输出 `row.rawContent`。

## 验证结果

- `rg` 静态确认：
  - `InviteStatsPage.tsx` 页面“原始消息”使用 `row.raw_content`。
  - `exportMemberTrace` 导出表头包含“原始消息”。
  - `exportMemberTrace` 导出行数据包含 `row.raw_content`。
- `git diff --check`：通过，仅提示 Git 未来可能按本地配置将 LF 转为 CRLF。
- `npm run typecheck`：未能完成，因为当前 `WeFlow-main/node_modules/.bin/tsc.cmd` 不存在，命令报 `tsc is not recognized`。本次未擅自安装依赖。

## 剩余风险

- 未实际生成导出文件做人工打开验证；原因是当前环境缺少运行应用和类型检查所需依赖。
- 如需完整端到端验证，需先安装 `WeFlow-main` 依赖，再启动应用导出一份群成员溯源 CSV / XLSX，确认最后一列为“原始消息”且内容非空。
