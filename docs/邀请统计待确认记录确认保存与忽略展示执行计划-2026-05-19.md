# 邀请统计待确认记录确认保存与忽略展示执行计划

## 目标

- 让待确认记录在 `WeFlow-main` 中形成清晰的人工处理结果。
- 点击“确认保存”后，记录进入数据大屏统计，并在群成员溯源中显示为“未退出群 / 有效”。
- 点击“忽略”后，记录仍能在群成员溯源中展示处理结果。

## 推荐口径

- `确认保存`：保留为确认后的有效入群记录，参与数据大屏、排行榜和溯源展示。
- `忽略`：建议作为独立处理结果保留，不建议直接伪装成真实退群。
- 如果业务必须显示成“已退出群”，建议仅作为展示层文案，不要误并入真实退群统计。

## 已确认口径

- `确认保存` 后的记录进入数据大屏统计，并在群成员溯源中显示为 `未退出群 / 有效`。
- `忽略` 是永久放弃处理，不并入真实退群语义。
- `忽略` 在群成员溯源中显示为 `已忽略 / 无效`。

## 执行步骤

1. 调整待确认记录确认保存链路。
   - verify: 确认后的记录进入大屏统计，并在溯源里显示为未退出群 / 有效。
2. 调整待确认记录忽略链路。
   - verify: 忽略后的记录仍能在溯源中展示，并按确认后的口径映射状态与归因。
3. 如有必要，调整导出文案与筛选逻辑。
   - verify: 页面、筛选、导出三处文案一致。
4. 补充本地变更记录。
   - verify: 本文件追加实际修改文件、验证结果、残余风险。

## 验证

- `WeFlow-main` 执行 `npm run typecheck`
- 手动验证待确认记录两条路径：
  - 确认保存后出现在大屏和溯源中。
  - 忽略后仍能在溯源中看到对应处理结果。

## 备注

- 本计划不涉及 `WeFlow-Web`。
- 本计划不修改 Supabase 表结构。

## 实施记录

- 当前实现已按已确认口径对齐：
  - `确认保存` 后的记录进入数据大屏统计。
  - 群成员溯源中确认后的入群记录显示为 `未退出群 / 有效`。
  - `忽略` 作为独立永久处理结果保留，溯源中显示为 `已忽略 / 无效`。
- 相关落点保持在主应用邀请统计页与邀请统计服务层，不引入表结构变更。
