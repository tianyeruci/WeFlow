# 邀请人数排行榜展示与全部活动口径对齐变更记录（2026-05-21）

## 已修改
- `WeFlow-main/src/pages/InviteStatsPage.tsx`
  - 邀请人数排行榜改为按条数自适应高度。
  - y 轴标签改为完整显示，不再截断或自动跳过。
  - 排行图表区域继续保留柱状图 / 饼图切换。
- `WeFlow-Web/lib/supabase-rest.ts`
  - 增加分页拉全的 Supabase 读取函数，避免单次查询触发默认上限。
- `WeFlow-Web/lib/invite-data.ts`
  - `final_stat_events` 和 `group_tag_bindings` 改为分页拉全。
  - dashboard 聚合改为按主端同样的 scope 关系过滤事件。
  - `group_tag_bindings` 按 `group_id` 去重并保留最新 `updated_at`。
  - `全部活动`、总成员数、今日新增、今日退群、排行榜、最近动态统一使用同一套范围。

## 原因
- 主端排行榜部分名字只隔行显示，根因是图表高度固定，ECharts 自动隐藏了部分 y 轴标签。
- Web 在“全部活动”下少算，根因是读取和聚合都没有完整覆盖全部行，并且没有和主端的绑定关系完全对齐。

## 风险与隐患
- Web 端分页拉全会比单次查询慢，但可以避免数据截断。
- 主端排行榜高度会随条数增长，数据量特别大时页面会更长，但展示更完整。
- 这次没有改表结构，也没有改同步链路，只是把读取和展示口径对齐。
