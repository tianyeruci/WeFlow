# 邀请统计功能调整变更记录

## 已完成

- WeFlow-main 待确认记录按钮改为“有效 / 无效”。
- 增加“同步本地数据”按钮，复用原有远端同步流程。
- 增加“检查是否退出群”按钮，并加入当前活动标签的后台退出检查。
- 增加北京时间每天凌晨的活动标签退出检查调度。
- 退出检查结果会影响本地统计，并通过扫描后同步回传远端。
- 群成员溯源列表补充头像展示。
- WeFlow-Web 群成员溯源列表补充头像展示。
- WeFlow-Web 统计口径补入“退出检查”导致的已退出群记录。
- WeFlow-Web 数据大屏“实时动态”改为优先显示成员真实头像，缺失时再显示首字占位。
- WeFlow-Web 发售群列表补充群头像展示，头像来源优先读取 `group_tag_bindings.raw_json.avatar_url/avatarUrl/head_img`。
- WeFlow-main 本地同步前会把群头像写入群标签绑定对象，后续同步到远端的 `group_tag_bindings.raw_json`，用于 WeFlow-Web 展示群头像。

## 验证

- `WeFlow-main`：`npm run typecheck`
- `WeFlow-Web`：`npm run typecheck`
- `WeFlow-main`：`npm run build`
- `WeFlow-Web`：`npm run build`

## 备注

- 目前按“有效/无效”按钮的语义保留原有确认流程，统计由现有有效记录口径接管。
- 退出检查以当前群成员列表为准，检测到缺失成员后会把对应入群记录标记为已退出群。
- 发售群列表的群头像不是新增表结构字段，而是复用已有 `raw_json`；部署新代码后，需要在 WeFlow-main 执行一次“同步本地数据”，远端才会拿到最新群头像。
