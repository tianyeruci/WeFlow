# WeFlow-Web 发售群列表：运营备注与人数排序执行计划

## Summary
- 只改 `WeFlow-Web` 发售群列表相关链路；不改 `WeFlow-main`，不读写微信原始群备注。
- 新增“运营备注”，按 `account_scope + group_id` 绑定，群名称变化后备注仍跟随同一个群 ID。
- 发售群列表新增“备注”列，支持行内编辑；仅“合计导出群列表”新增导出 `群备注`。
- 人数列支持升序/倒序排序，默认保持现有群列表顺序。
- 完成后保存变更记录到 `docs/发售群列表备注与人数排序变更记录-2026-05-26.md`。

## Key Changes
- 新增 Supabase 表 `web_group_remarks`：
  - 字段：`account_scope`、`group_id`、`remark`、`created_at`、`updated_at`。
  - 主键：`account_scope, group_id`。
  - SQL 脚本：`supabase/web_group_remarks_2026_05_26.sql`，包含建表、索引、注释和 `service_role` 权限。
- 新增环境变量 `REMOTE_REMARK_TOKEN`：
  - 仅备注写入接口校验该 token。
  - 普通查看、导出仍沿用当前公开读取逻辑。
  - 前端首次保存备注时弹窗输入 token，并存入 `localStorage`；后续保存自动带 `Authorization: Bearer ...`。
- 扩展数据接口：
  - `DashboardData.groups[]` 增加 `accountScope`、`remark`。
  - `getDashboard` 读取群绑定后，再按 `account_scope + group_id` 合并运营备注。
  - 新增 `PUT /api/invite/group-remarks`，写入 `{ accountScope, groupId, remark }`，清空备注时保存为空字符串。
- 调整发售群列表 UI：
  - 表头改为：`序号 / 群名称 / 备注 / 人数 / 操作`。
  - 备注列使用行内输入框，失焦或回车保存；保存中禁用当前行备注输入。
  - 保存失败保留用户输入并展示错误提示。
  - 人数表头增加排序按钮：默认无排序，点击后按人数倒序，再次点击切换升序。
  - 序号按当前显示顺序重新计算。
- 调整导出：
  - 仅 `mode=list` 的“合计导出群列表”新增 `群备注` 列。
  - CSV 表头为：`群名称, 群ID, 群备注, 人数`。
  - 如果页面当前按人数排序，导出群列表同步使用当前排序；未排序时保持现有默认顺序。

## Test Plan
- 在 `WeFlow-Web` 执行 `npm run typecheck`。
- 在 `WeFlow-Web` 执行 `npm run build`。
- 手动验证：
  - 输入正确 `REMOTE_REMARK_TOKEN` 后编辑备注，刷新页面后备注仍存在。
  - 清空备注后保存，页面和导出均为空备注。
  - 输入错误 token 时保存失败，并提示错误。
  - 切换活动标签后，备注按同一 `group_id` 保留。
  - 点击人数表头可在倒序、升序之间切换，序号同步变化。
  - “合计导出群列表”CSV 包含 `群备注`，其他群导出不新增备注列。

## Assumptions
- 一个群只会绑定一个活动标签；备注不按活动标签拆分。
- 运营备注只属于 `WeFlow-Web` 远端管理能力，不回写本地微信数据。
- 备注公开展示给能访问页面的人；只有修改备注需要 `REMOTE_REMARK_TOKEN`。
- 备注长度按前端限制为 300 字，后端保存前 `trim`，避免无意空格。
