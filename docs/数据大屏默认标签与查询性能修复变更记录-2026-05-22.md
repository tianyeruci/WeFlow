# 数据大屏默认标签与查询性能修复变更记录

日期：2026-05-22

## 本次目标

- 页面端数据大屏默认按“拉新”活动标签展示，首屏数据不再先按全部活动加载。
- 降低页面端大屏和群成员溯源远程查询压力，减少“远程数据暂不可用”和 502 风险。
- 群成员溯源列表支持分页，并在群筛选时把 `group_id` 下推到远程查询。
- 核对 WeFlow-main 当前排行榜默认口径，确认其已沿用不去重邀请溯源行计数逻辑。

## 已修改文件

- `WeFlow-Web/app/page.tsx`
  - `selectedTagId` 初始值改为空，标签加载完成后优先选择名称为“拉新”的活动标签，其次选择包含“拉新”的标签，再兜底第一个标签。
  - 活动标签下拉列表优先展示默认标签，之后才展示“全部活动”。
  - `loadDashboard` / `loadTrace` 在活动标签未就绪时不发起远程请求，避免首屏按全部活动误加载。
  - 群成员溯源请求增加 `limit` / `offset`，页面新增上一页/下一页分页。
  - 切换活动标签、群筛选、关键词、时间、状态、归因筛选时，同步回到第一页。

- `WeFlow-Web/app/api/invite/member-trace/route.ts`
  - 接收并限制 `limit` / `offset` 参数，默认每页 200 条，单页最大 500 条。

- `WeFlow-Web/lib/invite-data.ts`
  - `getMemberTrace` 支持分页返回 `limit` / `offset` / `hasMore`。
  - 群成员溯源事件读取支持传入 `groupId`，远程查询 `invite_events` / `quit_events` 时增加 `group_id` 条件。
  - `loadFinalEvents` 优先读取 `invite_events` / `quit_events`，仅在兼容表无数据时回退 `final_stat_events`，避免每次都查询旧统计视图。
  - 群成员溯源的群选项改为来自活动标签绑定关系，保证下拉选项与当前活动标签对应群一致。

- `WeFlow-Web/types/invite.ts`
  - `MemberTraceData` 增加 `limit`、`offset`、`hasMore` 字段。

- `docs/数据大屏默认标签与查询性能修复执行计划-2026-05-22.md`
  - 保存了本轮执行计划。

## 已核对

- WeFlow-main 当前 `buildInviteRanking` 默认 `dedupeMembers=false`，不去重时按邀请溯源行计数，只排除 `direct`。
- WeFlow-main 去重时才要求 `status=confirmed`、`valid_flag=1`、未退出且有被邀请人 wxid。
- WeFlow-main `getDashboard` 默认 `dedupeMembers` 仅在传入 `true` 时启用，符合“Main 默认与 Web 一致，Main 额外保留去重开关”的要求。

## 验证结果

- `WeFlow-Web`: `npm run typecheck` 通过。
- `WeFlow-main`: `npm run typecheck` 通过。
- 根目录：`git diff --check` 通过，仅输出 Git 对 CRLF/LF 的换行符提示。

## 仍需上线后手工确认

- 首次打开页面端数据大屏时，请确认 `/api/invite/dashboard` 首次请求的 `tagId` 是“拉新”活动标签 id，而不是 `__all__`。
- 在群成员溯源页选择单个群后，请确认接口请求带有 `groupId`，且列表分页正常。
- 生产环境 502 需要结合部署后的接口耗时和服务器日志确认；本次已减少默认全量查询和旧统计视图查询压力。
