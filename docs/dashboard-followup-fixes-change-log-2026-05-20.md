# 数据大屏与邀请统计后续修复变更记录（2026-05-20）

## 已修改内容

1. WeFlow-main 远程同步设置的“恢复初始化”确认流程
   - 原问题：点击恢复初始化后使用浏览器原生 `window.confirm`，弹框标题跟着窗口名显示为 `weflow`，无法定制。
   - 修改方式：改成页面内自定义确认弹窗，标题固定为“恢复初始化”。
   - 额外调整：确认按钮、遮罩关闭和取消按钮都接入了同一套状态控制。

2. 群人数展示进度条
   - 原问题：进度条满格基准只看当前最大群，导致 199 人看起来接近满格。
   - 修改方式：WeFlow-main 和 WeFlow-Web 都改为 `max(500, 当前范围最大群人数)` 作为满格基准。
   - 结果：未超过 500 人时，500 视为 100%；超过 500 时，按实际最大值归一。

3. 邀请人数排行榜下载图片按钮布局
   - 原问题：Web 端下载图片按钮所在工具栏过挤，视觉上不协调。
   - 修改方式：收紧 `WeFlow-Web` 的排行榜工具栏列宽，给下载图片按钮单独设置更稳定的宽度和不换行样式。
   - WeFlow-main 同步调整了排行榜工具区的对齐方式，避免控件高低不齐。

4. 邀请人数排行榜颜色展示
   - 原问题：WeFlow-main 的柱形图只有单一颜色，和 Web 端不一致。
   - 修改方式：WeFlow-main 柱形图改成和 Web 一致的多色柱体配置。

5. 排行榜说明文案
   - 原问题：“有效入群人数”措辞不符合当前口径。
   - 修改方式：WeFlow-main 和 WeFlow-Web 都改为“入群人数”。

6. 补充有效邀请的复制按钮
   - 原问题：复制按钮文字太重，视觉上像正文，不够干净。
   - 修改方式：WeFlow-main 改成图标按钮，仅保留 `title` / `aria-label`。

7. 检查是否退出群定时任务
   - 原问题：自动定时频率仍是 30 分钟。
   - 修改方式：改为启动后 1 分钟首次执行，之后每 1 小时执行一次；手动按钮保留。

8. WeFlow-main 发售群列表“全部活动”
   - 原问题：选择“全部活动”时，列表会漏掉只保存在绑定表里、但当前实时群列表没返回的群。
   - 修改方式：`listGroups` 结果里补上这些历史绑定群。
   - 结果：全部活动口径更完整。

## 已验证

## 剩余风险

- `WeFlow-main` 里仍存在其他页面的原生确认框，它们不是这次修复范围。
- 发售群列表补齐的是“历史有效绑定群”，如果实时聊天列表里缺头像或群名，会使用绑定表中的最后缓存值。
- 构建日志里仍有既有的体积警告，属于当前工程现状，不是这次改动引入的功能错误。
