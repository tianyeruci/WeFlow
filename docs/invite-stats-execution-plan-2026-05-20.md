# 邀请统计改造执行计划
日期：2026-05-20

## 目标

1. 保留现有表结构，只改行为和展示。
2. `WeFlow-main` 负责本地统计、定时任务和可交互改动。
3. `WeFlow-Web` 继续只读 Supabase，不接本机同步接口，不新增本机触发按钮。
4. 默认进入“全部活动”口径；`WeFlow-main` 的数据大屏增加“是否去重”开关，`WeFlow-Web` 不显示该开关。

## 关键假设

1. “全部活动”按当前账号下所有已启用群绑定的活动标签做汇总。
2. “去重”按 wxid 处理，全部活动场景下保留时间最早的一条。
3. “不去重”展示原始邀请数据，不再按状态或归因做额外收窄。
4. 待确认中的“被邀请人”复制按钮只复制当前显示昵称。
5. 定时任务改为：首次启动后 1 分钟执行，之后按固定间隔循环。

## 执行步骤

1. 服务层改造
   - 调整本地同步定时器：首次 1 分钟，之后每 3 分钟。
   - 调整退出群检查：首次 1 分钟，之后每 30 分钟。
   - 让“全部活动”成为可识别的统计范围。
   - 增加 dashboard 的去重参数，支持单活动 / 全部活动两种范围。
   - 修正同源重复确认时的合并策略，避免昵称漂移导致重复报错。

2. `WeFlow-main` 页面改造
   - 默认选中“全部活动”。
   - 增加“是否去重”开关，默认关闭。
   - 在待确认弹窗的“被邀请人”旁增加复制按钮。
   - 维持现有“添加信息”入口不变。

3. `WeFlow-Web` 页面改造
   - 默认选中“全部活动”。
   - 只读展示全量结果，不显示去重开关。
   - 维持 Supabase 直读，不接本机同步。

4. 验证与留痕
   - 跑两个项目的 `typecheck` 和 `build`。
   - 保存一份本次实际改动清单到本地。

## 验证标准

1. 启动后 1 分钟能看到首次同步和首次退出群检查。
2. `WeFlow-main` 里切换“全部活动 / 单活动”和“去重 / 不去重”时，统计口径稳定。
3. 待确认弹窗复制按钮可直接复制昵称到剪贴板。
4. `WeFlow-Web` 默认展示全部活动，且不出现去重按钮。
5. 两个项目构建通过。
