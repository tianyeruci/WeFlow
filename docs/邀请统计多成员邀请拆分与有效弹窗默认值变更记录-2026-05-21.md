# 邀请统计多成员邀请拆分与有效弹窗默认值变更记录

日期：2026-05-21

## 修改内容

- `WeFlow-main/src/pages/InviteStatsPage.tsx`
  - 待确认记录点击“有效”打开“补充有效邀请”弹窗时，“被邀请人微信 ID”默认使用已有 `row.wx_id`。
  - 如果 `row.wx_id` 为空，则默认填入被邀请人昵称 `row.user`，输入框仍可人工修改。

- `WeFlow-main/electron/services/inviteStatsService.ts`
  - 加固普通邀请系统消息匹配规则，允许邀请人和被邀请人两侧存在一个或多个英文引号。
  - 覆盖 `"A"邀请"B、C、D、E"加入了群聊` 和 `"A"邀请"B、C、D、E""加入了群聊` 这类多成员邀请消息。
  - 多成员拆分继续复用现有 `splitInvitedUsers`、`buildInviteEvent`、`hasInviteEvent`、`upsertRawEventFromInvite` 链路。

## 验证结果

- 已用本地解析样例验证：
  - `"A"邀请"B、C、D、E"加入了群聊` -> A 邀请 B/C/D/E 四条结果。
  - `"A"邀请"B、C、D、E""加入了群聊` -> A 邀请 B/C/D/E 四条结果。
  - `A邀请B,C，D、E加入群聊` -> A 邀请 B/C/D/E 四条结果。
- 已执行 `npm run typecheck`，结果通过。

## 残余风险

- 本次只影响新扫描进入的数据，不自动修复本地已扫描历史记录。
- 未在真实 Electron UI 中手动点击弹窗验证，需在应用内用待确认记录再做一次人工确认。
