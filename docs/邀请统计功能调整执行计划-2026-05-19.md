# 邀请统计功能调整执行计划

## 目标

1. 待确认记录只保留“有效 / 无效”两个处理入口，移除“保存确认 / 忽略”的旧表达。
2. 在 `WeFlow-main` 的“增量扫描”后增加“同步本地数据”按钮，异步触发本地数据到远端数据库的同步。
3. 在 `WeFlow-main` 和 `WeFlow-Web` 的群成员溯源中补充成员头像展示。
4. 增加“检查是否退出群”能力，支持当前活动标签下的群成员退群扫描，并支持北京时间凌晨定时扫描。
5. 同步本地数据时，确保由退群扫描产生的数据变更也会一并同步到远端。

## 需要确认的口径

- 群成员溯源中的数据改动，是否都要影响统计口径？
  - 当前默认假设：是，群成员溯源就是统计原始口径的一部分，改动后应立即影响大屏、排行和导出。

## 执行步骤

1. 梳理现有邀请统计链路与数据口径
   - verify: 明确待确认记录、群成员溯源、增量扫描、远端同步四条链路的输入输出与状态字段。
2. 调整 `WeFlow-main` 的待确认记录交互
   - verify: 待确认记录只显示“有效 / 无效”两个按钮，旧的“保存确认 / 忽略”不再出现。
3. 在 `WeFlow-main` 增加本地同步按钮
   - verify: 点击“同步本地数据”后异步触发现有本地到远端同步流程，界面不阻塞。
4. 增加“检查是否退出群”扫描
   - verify: 只扫描当前活动标签下已绑定的群；扫描后可将对应成员记录标记为已退出群，并重算相关统计。
5. 补齐 `WeFlow-main` / `WeFlow-Web` 的成员头像展示
   - verify: 群成员溯源列表中的成员项均显示头像，占位兜底正常。
6. 让退群扫描结果参与同步
   - verify: 退群扫描产生的状态变更会进入本地脏数据并随同步流程写入远端。
7. 补充变更记录
   - verify: 本文件追加最终修改范围、验证结果和剩余风险。
