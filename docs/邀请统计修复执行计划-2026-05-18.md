## Statistics And Flags

- 所有大屏统计都基于当前活动标签对应群里的系统消息数据：邀请/入群和退群。
- 人数类统计按活动标签范围去重，不再用实时微信群成员数兜底。
- 去重规则：
  - 原始消息：`account_scope + dedup_key`。
  - `dedup_key` 优先使用 `group_id + event_type + source_message_id + member`。
  - 没有 `source_message_id` 时用 `source_local_id + source_create_time`。
  - 再兜底用 `source_create_time + raw_content hash`。
  - 人员键优先 `wx_id`，没有时用规范化昵称。
  - 活动级人数按 `activity_tag_id + memberKey` 去重。
  - 群级人数按 `group_id + memberKey` 去重。
  - 今日退群按 `activity_tag_id + memberKey` 去重。

## valid_flag 含义

- `valid_flag` 表示“活动级统计有效性”，不是退群状态。
- `valid_flag = 1`：该入群记录参与当前活动标签人数统计和邀请人归因。
- `valid_flag = -1`：该入群记录保留在溯源明细，但不参与统计和归因。
- 退群状态看 `delete_flag` 或退群事件本身。
- `delete_flag = 1` 表示该条入群记录对应的单群关系已退出。
- 同一成员在同一活动标签下多次入群，只保留最早有效入群记录为 `valid_flag = 1`。
- 成员从该活动标签下所有群都退出后，原有效入群记录改为 `valid_flag = -1`。
- 退群事件自身不参与邀请归因，兼容数据里的 `valid_flag = 0/null` 视为“不适用”。

### 口径记录

- `valid_flag` 是活动级统计有效性：
  - `1` 参与当前活动标签人数统计和邀请归因。
  - `-1` 保留明细但不参与统计和归因。
  - 退群事件本身不参与邀请归因，兼容数据中的 `0/null` 视为不适用。
- `delete_flag` 是单群关系状态：
  - `1` 表示该条入群记录对应的单群关系已退出。
  - 非 `1` 表示该条入群记录对应的单群关系仍未退出。
- 状态筛选：
  - 未退出群：确认入群记录且 `delete_flag !== 1`。
  - 已退出群：退群事件，或入群记录 `delete_flag = 1`。
  - 待确认：解析状态为 `pending`。
- 归因筛选：
  - 有效：`valid_flag = 1`。
  - 无效：`valid_flag = -1`。
  - 待确认：解析状态为 `pending`。
