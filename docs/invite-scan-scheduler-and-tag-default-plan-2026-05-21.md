# WeFlow 活动标签与定时扫描调整计划（2026-05-21）

## Summary
- `WeFlow-main` 发售群列表不展示“全部活动”，但仍保留“聚合全部标签群”的展示结果。
- `WeFlow-main` 退出群检查改为启动后第 5 分钟触发一次，之后每 1 小时一次，手动触发保留。
- `WeFlow-main` 增量扫描改为启动后第 3 分钟触发一次，之后每 30 秒一次，扫描范围仍为所有活动标签对应的群。
- `WeFlow-Web` 数据大屏默认选中“拉新”标签。

## Key Changes
- 发售群列表：
  - 仅该页隐藏“全部活动”选项。
  - 聚合视图继续可用，但不把它包装成一个可选标签文案。
- 定时任务：
  - 退出群检查：首次延迟 5 分钟，后续 1 小时。
  - 增量扫描：首次延迟 3 分钟，后续 30 秒。
  - 增量扫描依旧扫描所有启用活动标签下已绑定的群，重复规则不变。
- Web 默认标签：
  - 优先选择名称为“拉新”的标签。
  - 若不存在“拉新”，回退到现有默认逻辑。

## Test Plan
- `WeFlow-main`
  - 发售群列表下拉不出现“全部活动”。
  - 启动后第 5 分钟触发退出群检查，之后每小时一次。
  - 启动后第 3 分钟触发增量扫描，之后每 30 秒一次。
- `WeFlow-Web`
  - 数据大屏默认显示“拉新”。
  - 无“拉新”标签时仍可正常打开并回退到原默认值。
- 验证命令
  - `WeFlow-main`: `npm run typecheck`、`npx vite build`
  - `WeFlow-Web`: `npm run typecheck`、`npm run build`

## Assumptions
- “拉新”按标签名称精确匹配。
- 发售群列表隐藏“全部活动”仅影响展示，不改变聚合视图的业务含义。
- 手动“增量扫描”和“检查是否退出群”按钮逻辑不改。
