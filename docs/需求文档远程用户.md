# WeFlow 邀请统计远程用户版需求文档

## 1. 背景

当前 WeFlow 已在本地完成邀请统计能力，可以读取本地微信系统消息，生成活动标签下的入群、邀请、退群和成员溯源数据。由于本地 WeFlow 具备扫描、标签管理、待确认修复等会影响统计结果的能力，不适合直接给普通用户远程观看。

远程用户版用于给普通用户查看邀请统计结果。普通用户只访问远程应用和远程数据库，不直接访问本地 WeFlow，不读取微信本地数据，也不具备任何会改变统计数据的操作权限。

## 2. 总体目标

- 提供一个可以部署在 Vercel 或类似平台的远程观看页面。
- 页面数据来自远程数据库，远程数据库由本地 WeFlow 扫描后同步。
- 普通用户只查看统计结果，不允许扫描、编辑、修复、确认、忽略或删除数据。
- 保持现有邀请统计页面整体视觉布局，不重新设计主信息架构，只针对隐藏功能后的空白区域做微调。

## 3. 架构说明

远程用户版采用以下架构：

```text
本地 WeFlow
  -> 扫描本地微信系统消息
  -> 同步原始消息和机器解析结果到远程 API

远程 API / Web 应用
  -> 部署在 Vercel 或类似平台
  -> 读取远程数据库
  -> 提供普通用户只读页面

远程数据库
  -> Supabase / Neon / Vercel Postgres 等
  -> 作为邀请统计唯一事实源
```

Vercel 只负责页面和 API，不依赖本地文件系统，不保存长期数据。所有持久化数据必须放在远程数据库中。

## 4. 权限范围

普通用户为只读角色，允许：

- 查看数据大屏。
- 查看群成员溯源。
- 使用页面内只读筛选条件。
- 查看 wxid、原始消息和溯源明细。
- 导出允许开放的统计数据。

普通用户不允许：

- 新增、编辑、删除活动标签。
- 设置、修改、清除群活动标签。
- 执行增量扫描。
- 执行全局扫描。
- 查看发售群列表页面。
- 查看待确认记录页面。
- 确认、忽略、拆分或修复待确认记录。
- 修改邀请人、被邀请人、wxid、状态、归因或有效性。

## 5. 页面范围

### 5.1 顶部导航

普通用户版只展示：

- 数据大屏
- 群成员溯源

不展示：

- 发售群列表
- 待确认记录

导航整体样式沿用当前邀请统计大屏风格。隐藏页面后，导航区域需要重新居中或按现有间距排列，避免出现明显空位。

### 5.2 数据大屏

数据大屏保留现有展示数据：

- 活动标签选择。
- 核心指标卡。
- 进群时段分布。
- 邀请人数排行榜。
- 群人数展示。
- 实时动态。
- 排行榜群筛选。
- 排行榜时间段筛选。
- 排行榜图表切换。
- 允许开放的导出功能。

数据大屏隐藏：

- 活动标签新增。
- 活动标签编辑。
- 活动标签删除。
- 增量扫描。
- 全局扫描。
- 扫描状态操作按钮。
- 任何会改变数据的入口。

活动标签选择为只读数据筛选，不提供管理能力。

### 5.3 群成员溯源

群成员溯源保留：

- 活动标签选择。
- 群选择。
- 成员昵称筛选。
- 时间段筛选。
- 状态筛选。
- 归因筛选。
- 含退群筛选。
- 明细表格。
- 原始消息查看。
- 允许开放的导出功能。

群成员溯源隐藏：

- 活动标签新增、编辑、删除。
- 增量扫描。
- 全局扫描。
- 待确认修复操作。
- 任何保存或修改按钮。

## 6. 数据口径

普通用户页面不直接读取原始系统消息表进行统计。所有统计结果必须来自最终统计视图。

数据分层如下：

- 原始消息层：保存微信系统消息凭据，不直接参与统计。
- 机器解析层：保存机器从原始消息中解析出的候选统计明细。
- 人工修正层：保存管理员修复、拆分、确认、忽略后的结果。
- 最终统计视图：页面展示和导出统一读取该视图。

最终统计视图规则：

- 如果某条原始消息存在人工修正结果，使用人工修正后的统计明细。
- 如果没有人工修正结果，使用机器确认的统计明细。
- 待确认记录不参与数据大屏、排行榜、成员人数和有效归因。
- 忽略记录不参与统计。

## 7. 待确认记录对普通用户的影响

普通用户不看到待确认记录页面，但待确认数量可以作为大屏指标展示。

待确认记录未处理前：

- 不进入邀请排行榜。
- 不进入今日新增。
- 不进入总成员有效统计。
- 不进入有效归因统计。

管理员处理后：

- 如果一条原始消息被拆分成多条统计明细，多条明细立即进入最终统计视图。
- 数据大屏、排行榜和成员溯源刷新后展示修正后的结果。

## 8. 远程部署要求

远程用户版必须满足：

- 可以部署到 Vercel 或类似 Serverless 平台。
- 不依赖本地 WeFlow 进程在线才能打开页面。
- 不依赖服务器本地文件。
- 数据从远程数据库读取。
- API 需要鉴权，普通用户 token 只能访问只读接口。

## 9. 接口权限建议

普通用户 token 允许调用：

- `GET /api/invite/tags`
- `GET /api/invite/dashboard`
- `GET /api/invite/member-trace`
- `GET /api/invite/export/ranking`
- `GET /api/invite/export/member-trace`

普通用户 token 禁止调用：

- 标签写接口。
- 群绑定写接口。
- 扫描接口。
- 待确认处理接口。
- 原始事件修改接口。
- 人工修正接口。

## 10. 验收标准

- 普通用户远程页面只出现数据大屏和群成员溯源。
- 页面中不存在增量扫描、全局扫描、活动标签新增、编辑、删除入口。
- 普通用户无法访问发售群列表和待确认记录。
- 普通用户筛选和导出读取的是最终统计视图。
- 待确认未处理前不影响统计展示。
- 管理员修复后，普通用户刷新页面能看到修正后的统计结果。
- 页面可以部署在 Vercel，并通过远程数据库读取数据。
