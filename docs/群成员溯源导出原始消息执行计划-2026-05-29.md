# 群成员溯源导出补充原始消息执行计划

日期：2026-05-29

## 需求理解

WeFlow-main 的“群成员溯源”页面已经能看到“原始消息”，但导出的群成员溯源文件缺少该列。目标是在不改变页面展示和筛选行为的前提下，让本地导出的 CSV / XLSX 文件包含同一条记录的原始消息。

## 关键假设

- 页面中的“原始消息”来源是 `InviteMemberTraceRow.raw_content`。
- 本次问题集中在 WeFlow-main 的本地导出链路，即 `window.electronAPI.inviteStats.exportMemberTrace` 到 `InviteStatsService.exportMemberTrace`。
- WeFlow-Web 远程导出链路已经包含 `rawContent`，本次不扩散修改到远程页面或 API，除非验证时发现同类缺口。

## 影响面

- 主要文件：`WeFlow-main/electron/services/inviteStatsService.ts`
- 预期改动：在 `exportMemberTrace` 的表头和每行数据末尾补充“原始消息”列，写入 `row.raw_content`。
- 不修改数据扫描、页面列表、筛选条件、待确认逻辑、原始事件导出逻辑。

## 执行步骤与验证

1. 确认页面展示字段与导出字段差异。
   verify：`InviteStatsPage.tsx` 表格使用 `row.raw_content`，`exportMemberTrace` 当前没有输出该字段。

2. 最小修改导出列。
   verify：`exportMemberTrace` 的 `headers` 包含“原始消息”，`body` 对应位置包含 `row.raw_content`。

3. 做静态验证。
   verify：运行 `npm run typecheck`，确认 TypeScript 类型检查通过或记录已有非本次问题。

4. 检查 diff 和改动边界。
   verify：`git diff --stat` / `git diff --check` 只反映计划文档、导出字段修复和最终变更记录。

5. 保存变更记录。
   verify：在 `docs` 新增变更记录文件，说明修改内容、验证结果和剩余风险。

## 完成标准

- 群成员溯源导出的 CSV / XLSX 都包含“原始消息”列。
- 该列内容来自页面同源数据 `row.raw_content`。
- 没有改变现有筛选、排序、分页、状态/归因文案逻辑。
