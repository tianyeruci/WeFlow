# 邀请人数排行榜统计口径与多微信合并变更记录

## 已完成
- 已保存执行计划：`docs/邀请人数排行榜统计口径与多微信合并执行计划-2026-05-22.md`。
- 调整 Main/Web 邀请人数排行榜口径：
  - 不去重时统计当前标签、群、时间范围内所有明确邀请人产生的邀请/扫码入群明细，不再受退出状态、`valid_flag` 和归因有效性限制。
  - 去重时按 `status=confirmed`、`valid_flag=1`、`delete_flag != 1` 且有成员标识的数据统计。
  - `direct` 直接入群仍不进入邀请人排行榜。
- 新增邀请人多微信合并能力：
  - Supabase 新增 `inviter_identity_mappings` 表和索引。
  - Web 大屏读取该表并把同一 `person_key` 下的多个 wxid 合并成一行排序展示。
  - Main 同步成功后从 Web 新增的 `/api/invite/inviter-mappings` 接口拉取映射，并缓存到本地统计数据中参与排行榜聚合。
- 调整总成员数默认口径：
  - Main 不去重时默认按当前标签下群成员数合计；去重且不含退群时尝试读取当前群成员 wxid 去重。
  - Web 不新增去重开关，默认按当前标签下群绑定的 `member_count` 合计展示。
- Web 邀请排行榜导出表头从“有效邀请人数”改为“邀请人数”，避免与不去重口径冲突。
- 远端恢复初始化逻辑未加入 `inviter_identity_mappings`，该后台维护表不会被清空。
