# 邀请统计标签管理与溯源状态修复

更新时间：2026-05-17

## 问题 1：活动标签管理入口不符合预期

现象：
- 顶部活动标签区域曾同时提供选择、编辑、删除和常驻的新建输入框。
- 用户预期不需要编辑活动标签，只需要能在“发售群列表”里创建和删除。
- 数据大屏、群成员溯源、待确认记录只需要活动标签选择、增量扫描、全局扫描。

解决方案：
- 移除活动标签编辑按钮和对应前端编辑逻辑。
- 保留删除能力，但只在“发售群列表”视图展示删除按钮。
- 新建活动标签改为点击“创建活动标签”按钮后弹出输入对话框，确认后创建，取消则不做任何操作。
- 数据大屏、群成员溯源、待确认记录的顶部工具栏只保留活动标签选择和两个扫描按钮。

## 问题 2：顶部工具栏布局不协调

现象：
- 常驻输入框占据过多横向空间，宽屏下显得松散。
- 编辑、删除、创建同时出现时，工具栏视觉重心偏乱。

优化方案：
- 将默认工具栏收敛为“活动标签选择 + 增量扫描 + 全局扫描”的紧凑布局。
- “发售群列表”增加轻量管理区，仅展示删除按钮和“创建活动标签”按钮。
- 固定扫描按钮宽度和高度，让顶部操作区在各视图之间保持稳定。
- 移除常驻新建输入框，降低顶部信息噪音。

## 问题 3：群成员溯源状态语义不正确

现象：
- 表格状态列展示的是解析确认状态，例如“已确认 / 待确认”。
- 用户实际需要的是群成员业务状态：未退出群、已退出群、待确认。
- 状态筛选选择“未退出群 / 已退出群 / 待确认”时，表格展示语义不一致。

解决方案：
- 表格状态列改为业务状态：
  - `status !== confirmed`：待确认。
  - `event_type = quit` 或 `delete_flag = 1`：已退出群。
  - 其他已确认入群记录：未退出群。
- 后端状态筛选同步按同一业务语义过滤。
- 成员溯源导出里的状态字段也同步使用“未退出群 / 已退出群 / 待确认”。

## 问题 4：原始消息展示不完整

现象：
- 原始消息列使用单行省略，较长系统消息无法完整查看。

优化方案：
- 表格中改成两行预览，避免撑高列表太多。
- 保留 `title` 悬停提示。
- 增加“查看”按钮，点击或双击原始消息单元格后弹出完整原始消息弹窗。
- 弹窗中使用自动换行和滚动区域，适合查看长系统消息。

## 问题 5：创建活动标签点击无效

现象：
- “创建活动标签”按钮点击后没有弹出输入框，用户无法输入新标签名称。

原因：
- 前端创建逻辑依赖 `window.prompt`，在当前 Electron 渲染环境里原生 prompt 可能不会可靠弹出，导致点击后看起来无响应。

解决方案：
- 改为页面内自定义“创建活动标签”弹窗。
- 点击“创建活动标签”后弹出输入对话框，输入名称后点击“确认创建”才会调用 `saveActivityTag`。
- 点击“取消”、关闭按钮或弹窗遮罩时只关闭弹窗，不创建标签。
- 创建过程中禁用输入和按钮，避免重复提交。

## 验证结果

- 已执行 `npm run typecheck`，结果通过。
- 本次改动范围集中在邀请统计页面、样式和活动标签管理交互。

## 影响范围

- 删除活动标签能力会清理该标签相关统计痕迹，但保留其他活动标签数据。
- 活动标签编辑能力已按最新预期从页面移除。
- 只有“发售群列表”承担活动标签创建和删除管理；其它页面只做选择与扫描。
- 群成员溯源“状态”列从解析确认状态调整为业务状态；解析确认状态不再在该列直接展示。
